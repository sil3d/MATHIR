MATHIR V7.1's Hybrid BM25+Dense+Cross-Encoder approach achieves 45.7% retrieval quality — a +14.1 percentage point improvement over pure dense retrieval (FAISS at 31.6%) on a real textbook corpus (White's Fluid Mechanics, 885 pages, 200 chunks, 50 ground-truth queries).

📚 Understanding the Results

Approach D's +14.1pp quality gain comes from combining three signals: (1) BM25 captures exact term matches that dense retrieval misses (e.g., "Reynolds number", "Navier-Stokes"), (2) Dense similarity captures semantic relatedness, and (3) Cross-encoder reranking re-evaluates top candidates with joint query-document attention.

The 64-dim projection in V7's default path loses information — projecting 384-dim to 64-dim drops quality from 31.6% to 19.7%.

  • Score Preservation: 100% — cached results return identical scores

💡 Key Insight

The cache provides massive speedup on repeat queries (158×) but no benefit on novel queries. This is expected — the cache stores (query, result) pairs, so identical queries benefit most.

Chat workloads excel because users frequently ask follow-up questions or rephrase the same query. In autonomous driving, cache hit rates exceed 90% because the vehicle repeatedly encounters the same situations.

🔬 Compression Methods

1

Sparse Coding (8× reduction)

ISTA algorithm compresses 272-dim vectors to 8-element sparse codes using learned dictionary.

2

TurboQuant 3-bit (~10.7× reduction)

Data-oblivious Hadamard rotation + scalar quantization. Per-coordinate optimal.

3

Combined Effect

1000×272-dim embeddings: 1,088,000 B → 116,976 B

⚠️ Trade-off Note

Compression adds ~0.12ms latency overhead (6.4% increase). However, the memory savings enable edge deployment where VRAM is constrained. The Variational Memory tier requires ~1MB and exceeds the 60KB budget — it's gated by a hard switch in the router.

🔍 Analysis

Novel queries score lowest (8.33%) because they test generalization to unseen concepts. This is expected — MATHIR learns from experience, and novel queries represent out-of-distribution inputs.

Similar queries score highest (60.78%) — paraphrases of stored memories benefit from dense similarity matching. The immunological memory would flag novel queries as anomalies, potentially routing to a different response strategy.

💡 Deployment Recommendations

Chat assistants, real-time queries, low-latency requirements.

Recommended: Approach A (Raw Embedding)
  • 657 QPS throughput
  • 1.54 ms median latency
  • 31.6% quality
  • Enable cache for 6ms on repeats
📚 Batch / Offline / RAG

Offline processing, research queries, quality-critical tasks.

Recommended: Approach D (Hybrid BM25+CE)
  • 2 QPS throughput
  • 494 ms median latency
  • 45.7% quality (+14.1pp)
  • Pre-warm cache for speedup
⚡ Edge / Ultra-Low-Latency

Embedded devices, real-time control, ≤50K chunks.

Recommended: FAISS (raw embeddings)
  • 20,392 QPS throughput
  • 0.05 ms median latency
  • 31.6% quality
  • No online learning

🎯 Hybrid Strategy: Cascade Architecture (V8)

Use FAISS as L1 retriever in front of MATHIR. FAISS returns top-50 in 0.05 ms; MATHIR reranks the top-50 in 150 ms amortized. This cascade architecture gets you FAISS speed with MATHIR quality.
