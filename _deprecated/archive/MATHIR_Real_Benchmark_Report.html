What MATHIR Should Have Done

  1. Swap the embedder first. MATHIR V7.1 uses all-MiniLM-L6-v2 (0.6451). Just swapping to BGE-base-en-v1.5 gives +9.2pp nDCG@10 instantly.
  2. The "Hybrid" pipeline hurts more than it helps on this dataset. BM25 fusion adds noise; for SciFact (abstract-based), pure dense wins.
  3. The 44× latency penalty is unjustified. 83.9ms is 1Hz — useless for real-time. Simple FAISS is 4.2ms = 238Hz.
  4. No "memory plugin" features were tested (online learning, anomaly detection). These are MATHIR's theoretical advantages, but they didn't show up in static retrieval.
