v7.2.1 — Performance & Metrics Overhaul

Session: June 2026 — Hybrid CPU/GPU + Dashboard fixes

FIX

CPU metric showing 0% — process-wide measurement

Problem: psutil.Process().cpu_percent(interval=None) returns 0% on Windows when called from a sleeping thread. The metrics thread sleeps 500ms between calls, so the delta CPU time is always ~0, even though storage threads are actively working.

Root cause: cpu_percent() computes (cpu_time_delta / wall_time_delta). When the calling thread is the only one measured, and it's sleeping, the result is 0%.

Fix: Manual cpu_times() delta — measures the entire process (all threads), regardless of which thread calls it.

- if not self._cpu_primed: - cpu = process.cpu_percent(interval=0.1) - self._cpu_primed = True - else: - cpu = process.cpu_percent(interval=None) + cpu = self._get_cpu_percent() # process-wide, thread-independent # New method in MetricsCollector: + def _get_cpu_percent(self): + ct = self._process.cpu_times() + cpu_time = ct.user + ct.system # ALL threads + now = time.monotonic() + dt_wall = now - self._last_cpu_mono + dt_cpu = cpu_time - self._last_cpu_times + return (dt_cpu / dt_wall) * 100.0

FIX

GPU showing global VRAM instead of MATHIR-specific

Problem: nvidia-smi --query-gpu=memory.used returns total GPU memory used by all processes (Windows display driver, CUDA context, other apps). This is misleading for a MATHIR stress test.

Fix: Use torch.cuda.memory_allocated() which returns only tensors allocated by this PyTorch process.

- # PRIMARY: nvidia-smi (shows ALL GPU processes) - result = subprocess.run(["nvidia-smi", - "--query-gpu=memory.used,utilization.gpu", ...]) - vram_mb = float(parts[0]) # GLOBAL VRAM + # VRAM: MATHIR-specific via torch.cuda + vram_mb = torch.cuda.memory_allocated() / 1024 / 1024 + # Util: nvidia-smi (system-wide, only option) + util_pct = nvidia_smi_query("utilization.gpu")

FIX

Stress test resuming old data on restart

Problem: stress_memory.db persists on disk between runs. Clicking "Start" after "Stop" would show old conversations (3.5K from previous run) because the DB was never deleted.

Fix: start() now deletes stress_memory.db + SQLite WAL/SHM files before reinitializing.

+ # Clean slate: delete old DB before reinit + if os.path.exists(self._db_path): + os.remove(self._db_path) + for ext in ("-wal", "-shm"): + if os.path.exists(self._db_path + ext): + os.remove(self._db_path + ext)

FEATURE

System Health Bar — 3-color visual indicator

What: A color-coded bar showing overall system health. Green = optimal, Blue = moderate load, Red = needs attention.

Metrics scored: CPU %, GPU MB, Recall latency, Errors, DB write latency.

Architecture: Thresholds are server-driven, not hardcoded. Sent via WebSocket health_config event on connect and config update.

server.py stress.html | | | config["health_thresholds"] | | cpu: [30, 70] | | gpu: [200, 500] | | recall: [5, 20] | healthThresholds = { | errors: [0, 5] | cpu: [30, 70], | db_write: [10, 50] | gpu: [200, 500], | | ... +--- WS "health_config" ----------->+ } | scoreMetric(val, thresholds) | 0 = green (good) 1 = blue (normal) 2 = red (critical)

To modify thresholds: edit server.py → self.config["health_thresholds"]. Frontend receives updates on next connect.

v7.2.0 — Performance Optimizations

O(n) → O(1) algorithms, vectorized operations, thread safety

PERF

O(1) LIRS eviction in raw_episodic.py

Before: O(n) linear scan to find least-recently-used entry on every store().

After: O(1) using OrderedDict with LIRS (Low Inter-reference Recency Set) algorithm.

mathir_lib/memory/raw_episodic.py
PERF

O(n) centroid-based forget in ensemble_episodic.py

Before: O(n²) pairwise cosine similarity to find clusters for deletion.

After: O(n) by computing cluster centroids and removing nearest entries to centroid.

mathir_lib/memory/ensemble_episodic.py
PERF

Bounded BM25 + lazy cross-encoder in hybrid_episodic.py

Before: BM25 corpus rebuilt on every store(). Cross-encoder loaded at init.

After: BM25 corpus bounded at 5000 docs. Cross-encoder lazy-loaded on first use.

mathir_lib/memory/hybrid_episodic.py
PERF

Cholesky solve + cached inverse in immunological.py

Before: O(D³) Mahalanobis distance computed on every perceive(). Full matrix inverse cached.

After: Cholesky decomposition for stable solve. Inverse cached and invalidated on matrix change.

mathir_lib/memory/immunological.py
PERF

Vectorized scatter in semantic.py

Replaced element-wise numpy updates with vectorized scatter_add_ operations for faster semantic memory updates.

mathir_lib/memory/semantic.py
PERF

Vectorized numpy cosine + argpartition in store.py

Before: Python loop computing cosine similarity per vector.

After: Batch numpy cosine with argpartition (partial sort) — 10× faster for multi-query search.

v7.2.0 — Architecture

FEATURE

HybridDeviceManager + auto device_map

Added device_map="auto" to MATHIRPluginV7 and MATHIRMemory. Auto-detects CUDA/CPU and places all components on the best available device.

mathir_lib/hybrid_device.py mathir_lib/device_utils.py mathir_lib/plugin_v7.py mathir_dropin/memory.py
SECURITY

Thread safety — threading.RLock on all memory tiers

Added threading.RLock to 6 memory classes to prevent race conditions in multi-threaded stress testing.

mathir_lib/memory/working.py mathir_lib/memory/episodic.py mathir_lib/memory/semantic.py mathir_lib/memory/immunological.py mathir_dropin/memory.py
SECURITY

5 CRITICAL device-mismatch bugs fixed

In-place pointer updates that didn't account for GPU→CPU transfers. Fixed with proper .to(device) calls before operations.
